Few REST calls to let you explore the Todo backend app:

  • Create a tenant:
    curl -X POST 'http://localhost:8080/tenants' --header 'Content-Type: application/json' --data '{"name":"my first customer"}'
  • Take a look at the new tenant:
    curl  -X GET 'http://localhost:8080/tenants'
  • Create a todo for the tenant. Make sure you use the tenant ID you got previously in your URL: 
    curl -X POST 'http://localhost:8080/tenants/d4f8ce99-48ad-4275-8f21-885d1d7d9613/todos' --header 'Content-Type: application/json' --data '{"title": "feed the cat", "complete": false}'
  • Get the TODOs for the tenant: 
    curl  -X GET 'http://localhost:8080/tenants/018aaf79-51b0-77c5-b1a4-24180a9c3329/todos'
  • Repeat the previous steps to create another tenant with their own TODO
  • Now check the insecure endpoint that bypasses the tenant-aware interceptor and therefore sees TODO for all tenants: 
    curl  -X GET 'http://localhost:8080/insecure/all_todos'
